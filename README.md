# E-Commerce Analytics Pipeline (Olist Dataset)

This project demonstrates an end-to-end data analytics workflow similar to what real e-commerce companies use for business reporting and decision-making.

## Business Objective
To analyze sales performance, customer behavior, seller efficiency, delivery operations, and customer satisfaction using SQL and Power BI.

## Tech Stack
- Python (data cleaning & preprocessing)
- PostgreSQL (data storage & SQL analysis)
- Power BI (interactive dashboards)
- SQL (business queries & KPI calculations)

## Data Source
Brazilian E-Commerce Public Dataset by Olist (Kaggle).  
The dataset contains multiple tables such as orders, customers, products, sellers, payments, reviews, and geolocation.

## Project Workflow
1. Cleaned and preprocessed multiple raw datasets using Python.
2. Loaded cleaned data into PostgreSQL.
3. Designed SQL queries to answer real business questions.
4. Built interactive Power BI dashboards for stakeholders.

## Dashboard Pages
- Executive Summary
- Sales & Customer Insights
- Operations & Delivery Performance
- Reviews & Seller Performance

## Key KPIs
- Total Revenue
- Total Orders
- Average Order Value
- Revenue Growth (MoM)
- Repeat Customer %
- On-Time Delivery %
- Average Review Score

## Key Business Insights
- A small group of sellers contributes a large share of total revenue.
- Late deliveries have a clear negative impact on customer ratings.
- Repeat customers generate significantly higher revenue per customer.
